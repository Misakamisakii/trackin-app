# Trackin' - 音乐项目进度与版本管理平台 PRD

## 1. 产品概述

### 1.1 产品定位
Trackin' 是一款专为音乐人（制作人、混音师、母带工程师及独立音乐人）打造的专业级音乐项目制作进度追踪与音频版本控制平台。通过现代化的看板与甘特图结合的方式，解决音乐制作过程中“文件版本混乱”、“沟通反馈脱节”、“整体进度不透明”的三大痛点。

### 1.2 核心口号
> **Keep track of your track.**

### 1.3 目标用户
- 独立音乐人
- 音乐制作工作室
- 乐队成员（用于协作）

## 2. 核心功能需求

### 2.1 全局导航 (Global Navigation)
- **Logo 导航**：在除主页外的所有页面，点击左上角的 `Trackin'` 标题/Logo，均可直接返回到主页（Landing Page）。
- **面包屑导航 (Breadcrumbs)**：在工作区 (Workspace) 等深层级页面，采用 `Trackin' / [项目名称] / Workspace` 的面包屑格式替代传统的后退按钮。点击对应的面包屑层级可直接跳转至该前级页面（例如点击项目名称返回项目列表）。

### 2.2 主页 (Landing Page)
- **视觉基调**：浅绿色（Mint/Sage Green）主题，搭配极简黑色排版，营造清新、专注的创作氛围。
- **Hero Section**：
  - 居中展示带高级阴影效果的黑色巨型字体口号：“Keep track of your track”。
  - 核心 CTA 按钮：[Start tracking]（点击后跳转至一级目录：项目库）。
  - 底部视觉元素：动态流动的黑色音符波浪图案（CSS/SVG/Framer Motion 动画实现）。

### 2.3 一级目录：项目库 (Project List)
- **核心交互**：点击主页的 [Start tracking] 按钮后进入。
- **新建与编辑项目**：支持以弹窗 (Modal) 形式填写或修改项目信息并保存。属性包含：
  - **Cover Upload**：支持上传本地图片 (png/jpg/jpeg) 作为项目封面。
  - **Title**：项目名称
  - **Artist**：艺术家
  - **Category**：项目类型（下拉选择：Album / EP / Single / Beattape）
  - **Genre**：音乐风格
  - **Introduction**：项目简介/描述
- **项目列表**：以网格卡片形式展示已有项目。卡片左侧展示封面缩略图（若无则展示默认占位图），右侧显示项目属性。鼠标悬停时提供 **编辑 (Edit)** 和 **删除 (Delete)** 快捷操作按钮。点击具体项目卡片，即可进入该项目的二级目录（详情工作区）。

### 2.4 二级目录：项目工作区 (Project Workspace)
这是产品的核心工作区，包含以下功能模块：

#### 2.3.1 进度看板与元数据 (Progress & Metadata)
- **项目属性展示**：顶部区域以大标题+封面图形式展示，封面图位于标题文字右侧。下方卡片展示项目的描述 (Introduction) 等基本信息。
- **双视图切换**：提供 `Metadata` (元数据列表) 与 `Progress` (进度看板) 两种视图，支持无缝动画切换。
- **Metadata 列表**：
  - 支持随时 **新增单曲** 和 **删除单曲**。
  - **拖拽排序 (Drag to Reorder)**：支持拖动单曲左侧的握把 (Grip) 图标对单曲进行上下排序。序号变更后，Progress 视图中对应的单曲序号也会同步更新。
  - 展示并维护曲目基本信息，支持点击展开详情表单。
  - **所有信息实时可编辑**：包括曲名、演唱者、作词、作曲、采样源、BPM、风格等。
- **Progress 看板**：
  - 横向展示制作流程（Demo -> Arranging -> Recording -> Mixing -> Mastering -> Done）。
  - **拖拽交互 (Drag & Drop)**：支持通过直接拖动单曲模块到不同的列（Stage）来改变该单曲的进度状态。单曲卡片上会清晰展示来自 Metadata 列表的全局序号。状态改变会与 Metadata 视图实时同步。

#### 2.3.2 单曲详情与版本控制 (Track Version Control)
每首单曲独立的管理面板，核心包含：
- **元数据信息**：BPM、调性 (Key)、参考曲目 (Reference Track)、备注。
- **制作阶段流转**：Demo -> 编曲 (Arrangement) -> 录音 (Recording) -> 混音 (Mixing) -> 母带 (Mastering)。

##### 2.3.2.1 Version 版本树模块（新增）
**模块定位**：
- 模块名称：Version（版本树）
- 所属模块：Workspace
- 模块目标：用于对专辑中单曲的不同制作版本进行可视化管理，以"流程图/分支树"形式呈现版本演进路径，帮助用户清晰追踪创作迭代过程，支持版本回溯与对比。

**设计范式**：
参考飞书流程图工具的交互方式：
- 画布式（Canvas）操作，支持拖拽、缩放
- 节点 + 连线结构，分支式扩展（类似流程分叉）

**核心功能说明**：

1. **父节点自动生成（单曲初始化）**
   - **触发条件**：当用户在 Metadata 模块中新增一首单曲时
   - **系统行为**：自动在 Version 模块中生成一个对应的父节点（Root Node）
   - **节点位置**：固定在画布最左侧，每首单曲对应一棵独立的版本树
   - **节点默认信息**：
     - 版本名称：默认继承单曲名称（可编辑）
     - 主要修改点：空
     - 音频文件：初始 Demo 或空

2. **子节点创建（版本分支）**
   - **交互方式**：用户点击任意节点 → 出现「+ 添加版本」按钮（位于节点右侧）
   - **点击效果**：在该节点右侧生成子节点，并自动建立连线
   - **规则**：
     - 每个节点可创建多个子节点（多分支）
     - 所有子节点以横向分支结构展开
     - 不同分支之间视觉上需要区分（避免混淆版本路径）
   - **布局逻辑**：
     - 父 → 子：从左到右
     - 多分支：向右侧呈"树状扩散"
     - 自动避免节点重叠（基础自动布局）

3. **节点属性定义**
   每一个版本节点包含以下三个核心字段，均可随时编辑：
   - **版本名称**（String）：如 V1 Demo / V2 Mix / Final
   - **主要修改点**（Text）：描述该版本相较父版本的改动（如：加鼓、换和弦）
   - **音频文件**（File）：上传对应版本音频（支持播放）

**核心交互流程**：
- **创建版本路径**：用户进入 Version 模块 → 查看已有单曲的版本树 → 选择某一节点 → 点击「+」→ 填写版本信息 → 上传音频 → 生成新分支节点
- **分支管理**：支持从任意历史节点继续分叉（非线性创作），支持多个版本并行演进（如不同编曲方向）
- **节点操作（基础能力）**：
  - 编辑节点信息（名称 / 修改点 / 音频）
  - 删除节点（需确认，避免误删分支）
  - 节点点击可展开详情侧边栏

**可视化与体验要求**：
- **视觉结构**：
  - 节点：卡片式（类似流程图节点）
  - 连线：直线 or 曲线（清晰表达父子关系）
  - 当前选中节点高亮
- **音频体验（重点）**：
  - 节点内支持「播放按钮」
  - 支持快速对比（后续可扩展 AB Test 播放）

**约束与边界**：
- 每首单曲仅允许一个根节点
- 不支持环状结构（必须为树）
- 不支持跨单曲节点连接

**视图切换**：
- 提供「画布视图」和「列表视图」两种展示方式
- 画布视图：支持拖拽、缩放、树形结构展示
- 列表视图：传统的线性列表展示，便于快速浏览

## 3. 交互与体验设计原则 (基于 frontend-design skill)

- **极简主义与大留白**：拒绝信息过载，操作面板必须清爽。
- **高级排版**：使用无衬线几何字体（如 Space Grotesk 或 Plus Jakarta Sans）作为标题，搭配易读的正文字体。
- **拟物与扁平结合 (Neumorphism hints)**：在音频播放器和进度条上使用轻微的内阴影，增加可触控感。
- **流畅动效**：页面切换、播放器展开、版本切换必须有丝滑的过渡动画。
